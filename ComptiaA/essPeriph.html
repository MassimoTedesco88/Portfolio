[ Essential Peripherals ]

~ Central Process Unit ~

Definition:

the CPU is the brain of the computer, responsible for executing instructions and processing data.

Key CPU Components:

  • EDB
  • Register
  • Clock speed
  • System crystal (Quartz oscillator)
  • Address Bus
    •

EDB (External Data Bus): the pathway used to transfer data between the CPU and other components like RAM

Registers: small, fast storage locations inside the CPU used to hold data and instructions temporarily during processing. The four most common: general-purpose registers (AX, BX, CX, DX)

Clock speed: measured in Mhz or Ghz, indicating how many cycle per second the CPU can perform.

System crystal: determines the operating frequency of the CPU and the system by generating e consistent clock signal.

Address Bus: Pathways used by the CPU to communicate memory address to the memory controller and access specific locations in RAM. Memory Controller Chip (MCC) is a hardware component responsible for managing the flow of data between the CPU and system’s RAM.

Thermal design power (TDP)

is the maximum amount of heat a computer chip (like a CPU or GPU) is expected to generate undertypical workload conditions. It’s measured in watts and helps hardware designers choose appropriate cooling system to keep the chip operating safely and efficiently. (TDP is not the absolute maximum power the chip can use).

Smartphone, tablet TDP from 2 to 15 watts
Laptop CPUs TDP from 7 to 65 watts
Desktop CPUs TDP from 50 to 140 watts

Features:

  • Clock multipliers
  • 64-bit processing
  • Virtualization support
  • Parallel execution
  • Multicore processing
  • Integrated memory controller (IMC)
  • Graphics processing unit (GPU)
  • Security

Clock multipliers: a clock multiplier is a factor used to determine the operating frequency of a CPU based on the base clock speed of the system.

CPU speed = Base Clock x Multiplier

3.6GHz = 100MHz x 36

64-bit Processing: refers to a CPU architecture that handles data in 64-bit chunks.

This allows for:

  • more memory access (up to 18.4 million TB of RAM theorically)
  • better performance with large datasets
  • enhanced security features

x86: originally referred to Intel’s 32-bit processor architecture. Over time, it became a general label for 32-bit instruction set architectures used in PCs.

x64: 64-bit extension of x86.

X86-64:technical term for x64.

Virtualization Support: enables a CPU to run multiple operating systems simultaneously using virtual machines.

Parallel Execution: refers to the ability of a CPU to execute multiple instructions or tasks at the same time, rather than one after another.

Pipeling: is a technique where the CPU breaks down instruction execution into separate stages, allowing multiple instructions to be processed simultaneously – one in each stage.

This boosts instructions through put.

Stages:

  1. Fetch: get the data from the EDB
  2. Decode: figure out what type of command needs to be executed
  3. Execute: perform the calculation
  4. Write: send the data back on to the EDB

Cache: is a small, fast memory inside or near the CPU that stores copies of frequently used data from the main memory (RAM), helping the CPU access important information much faster.

Threads: a thread is the smallest unit of execution inside a program. Multiple threads can run parts of a program at the same time, allowing tasks to happen faster and more efficiently.

Wait states: a wait state is a short delay that occurs when the CPU has to wait for data from slower devices like RAM or storage. While waiting, the CPU temporarily pauses because it cannot continue processing without the required information.

L1, L2, L3 Cache

  • L1 Cache: Smallest and fastest cache, built directly into the CPU core. Stores critical data and instructions needed immediately.
  • L2 Cache: Larger than L1 Cache but a bit slower. Acts as a backup for L1, holding more data.
  • L3 Cache: Even larger and slower compared to L1 and L2. Shared across multiple cores to improve communication between them.

Multithreading: is the general concept where a CPU( or a program) handles multiple threads at the same time to improve performance.

Hyper-Threading: is Intel’s brand name for a special type of multithreading where one physical CPU core acts like two logical cores. It lets a single core process two threads simultaneously to better use CPU resources and increase speed.

Multicore processing: means a CPU has two or more independent cores inside a single chip. Each core can run its own tasks or threads at the same time, which makes the CPU much faster and better at multitasking compared ti a single-core processor.

Integrated memory controller (IMC): is a memory controller that is built directly into the CPU instead of being a separate chip on the motherboard. It manages communication between the CPU and the system’s RAM, making data transfer faster and more efficient by reducing latency.

Integrated graphics processing unit (iGPU): is a graphics processor built into the CPU (or sometimes into the motherboard chipset). It handles basic graphics tasks like video playback, webbrowsing, and light gaming without needing a separate (dedicated) graphics card.

Security: the NX Bit (No-eXectute Bit) is a hardware-based security feature found in Intel, AMD and ARM CPUs. It marks certain areas of memory a “non-executable”, meaning code cannot run from those areas. This helps prevent malware attacks (like buffer overflow attacks) by blocking malicious code from being executed in memory regions meant only for data.

Each company’s name for NX Bit:

  • Intel: XD bit (eXecute Disable bit)
  • AMD: NX bit (No eXecute bit)
  • ARM: NX bit (Execute Never bit)

Main Purpose:

  • Stops harmful code from running.
  • Increases system protection without slowing down performance.
