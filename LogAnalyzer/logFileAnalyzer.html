System Log Analyzer for Local Security

A lightweight Python tool developed to analyze system logs on Linux(Ubuntu), with the goal of detecting potential intrusions, unauthorized login attemps, and suspicious activity. The script inspects key security-related events by querying system logs via journalctl.

Note: The code in this project was written with the help of ChatGPT.

Technologies Used

Linux (journalctl), Python, shell scripting

Why I built it

I wanted to improve my understanding of Linux system auditing and learn how to programmatically detect early signs of unauthorized access or misuse. This project allowed me to explore log analysis and implement basic security monitoring techniques using native system tools.

**Torna Indietro**

What it does

  • Scans system logs from recent days using journalctl
  • Detects suspicious events such as:
    • Failed login attempts
    • Login attempts by non-existent users
    • Sessions started by unknown or untrusted accounts
    • Improper use of administrative privileges

What I learned

This project helped me improve my ability to analyze system logs and extract meaningful patterns related to security. I focused on identifying suspicious authentication behaviors and defining criteria to detect potential threats. It also strengthened my skills in scripting, log parsing, and designing simple rule-based detection logic.
